# Changelog

Все значимые изменения AvaTracker Mobile фиксируются в этом файле.

## [1.0.6+15] - 2026-07-22

### Added

- локальная оценка яркости, контраста и резкости кадров фронтальной камеры;
- тесты качества кадров для YUV/BGRA-потоков Android и iOS.

### Changed

- проверка лица после QR стала автоматической: камера сразу показывает превью,
  через одну секунду выбирает момент по яркости и резкости и отправляет один
  JPEG-снимок без ручного нажатия и без передачи видеопотока на backend;
- при отсутствии подходящего кадра приложение не задерживает сотрудника:
  снимок автоматически выполняется по короткому таймауту;
- при ошибке съёмки выполняется до трёх автоматических попыток.

### Verification

- `flutter analyze` — без ошибок;
- `flutter test` — 54 теста прошли;
- production APK `AvaTracker-v1.0.6.apk` собран с `MOCK_API=false`,
  `TEST_AUTH=false`, `versionName=1.0.6`, `versionCode=15`;
- SHA-256:
  `97575b19c9120980aec8607604a92b091f844d2bd17d6d970b159bde6e7dafa0`.

## [1.0.5+14] - 2026-07-20

### Added

- регистрация сотрудников Узбекистана по 14-значному ПИНФЛ (JShShIR),
  передаваемому в существующем поле `iin`;
- автоматическая смена языка: UZ выбирает узбекский, KZ — казахский.

### Changed

- UZ-телефон отправляется как 9 национальных цифр и `dial_code="+998"` во
  всех шагах регистрации, входа и восстановления пароля;
- Политика конфиденциальности заменена документом версии 2.0 из
  `Политика .docx`; `PRIVACY_POLICY.txt` стал единым asset-источником экрана.

### Fixed

- mock SMS-подтверждение корректно распознаёт полный номер `998…`;
- при смене страны очищаются телефон и идентификатор, исключая смешение
  форматов KZ и UZ;
- ПИНФЛ для Узбекистана принимает любое непустое цифровое значение до
  14 символов; ограничение ввода сохраняется. Проверка ИИН Казахстана на
  точные 12 цифр не изменилась.

### Verification

- `flutter analyze` — без ошибок;
- `flutter test` — 52 теста прошли;
- production APK `AvaTracker-v1.0.5.apk` собран с `MOCK_API=false`,
  `TEST_AUTH=false`, `versionName=1.0.5`, `versionCode=14`;
- `PRIVACY_POLICY.txt` версии 2.0 присутствует внутри APK.

## [1.0.4+13] - 2026-07-14

### Changed

- табель переведён с несуществующего timesheet-маршрута на реальный
  `/api/v1/employee-identification-list/`;
- клиент проходит все страницы отметок, группирует их по дням,
  считает первую отметку приходом, а последнюю — уходом;
- `auth_time` приводится из серверного часового пояса ко времени устройства;
- график дополняется из `/api/v1/employees/{iin}/`, а при отсутствии
  schedule-полей — из `/api/v1/employees/?search={iin}`.

### Fixed

- вкладки переведены на `StatefulShellRoute.indexedStack`: QR-экран и его
  `MobileScannerController` больше не пересоздаются при быстром переключении.
- табель передаёт в `employee-identification-list` исключающую верхнюю
  границу периода: отметки текущего дня больше не пропадают.

### Verification

- `flutter analyze` — без ошибок;
- `flutter test` — 49 тестов прошли;
- production APK `AvaTracker-v1.0.4.apk` собран с `MOCK_API=false` и
  `TEST_AUTH=false`; перед распространением нужна проверка на физическом
  устройстве.

## [1.0.3+12] - 2026-07-13

> Сборка отозвана: в ней был указан несуществующий production endpoint
> табеля, а исправление камеры не устраняло гонку при смене вкладок.

### Added

- отдельная секция «График работы» в аналитике с диапазоном
  из `schedule_start_time` и `schedule_end_time` карточки сотрудника;
- строки прихода и ухода в карточке каждого дня табеля.

### Changed

- табель переведён на `/api/v1/mobile/attendance/timesheet`; все отметки
  показываются независимо от наличия опоздания;
- текст источника опозданий уточнён для первой отметки
  относительно начала смены;
- mobile-профиль дополняется данными `/api/v1/employees/{iin}/`.

### Fixed

- запуск, остановка и освобождение QR-камеры выполняются
  последовательно; быстрое переключение вкладок больше не запускает
  два `MobileScannerController` одновременно.

### Verification

- `flutter analyze` — без ошибок;
- `flutter test` — 45 тестов прошли;
- production-конфигурация APK собрана с `MOCK_API=false` и `TEST_AUTH=false`;
- APK: `AvaTracker-v1.0.3.apk`, `versionName=1.0.3`, `versionCode=12`;
- SHA-256:
  `bb19679e7d07d1bf0a8071ef695ff84ecd10483b27b7f7baa8dea6e9b027d0b8`;
- локальный production keystore не настроен, поэтому APK пока подписан
  `CN=Android Debug` и предназначен только для внутреннего теста.

## [1.0.0+9] - 2026-07-08

### Added

- поддержка системного менеджера паролей через autofill hints:
  iOS Passwords/Keychain и Android Password Manager могут предложить
  сохранить телефон и пароль после успешного входа или регистрации;
- единая release-сборка `AvaTracker-v1.0.0.apk` для внутреннего тестирования.

### Changed

- первый intro-экран перед входом убран: неавторизованный пользователь сразу
  попадает на экран входа;
- клавиатура скрывается при нажатии вне поля ввода;
- приветствие на экране входа изменено на «Добро пожаловать!», версия
  приложения на экране входа скрыта;
- выбор языка в разделе «О приложении» приведён к форме выбора языка на
  экране авторизации;
- текст QR-экрана уточнён: «в радиусе 50 м»;
- во вкладке «Табель» удалён дублирующий блок опозданий за месяц;
- во вкладке «Аналитика» оставлен месячный режим и четыре равных показателя:
  случаи, суммарно, в среднем, максимальное опоздание;
- в профиле убраны дублирующая строка должности и организация, аватарная
  рамка заменена на чёрную;
- в разделе «О приложении» убран пункт согласия на обработку данных, диалог
  поддержки сокращён до одной кнопки «Написать письмо»;
- анимация переходов между страницами отключена.

### Fixed

- поле телефона больше не принимает цифры сверх лимита страны и не стирает
  первые введённые цифры;
- во вкладке «Табель» исправлена подсказка: опоздания отмечены оранжевым,
  а не красным;
- тексты удаления аккаунта сокращены, чтобы убрать дублирование.

### Security

- приложение по-прежнему не хранит пароль локально: сохранение пароля отдаётся
  системному менеджеру паролей устройства;
- password fields отключают автокоррекцию и текстовые подсказки клавиатуры.

### Verification

- `flutter analyze` — без ошибок;
- `flutter test` — 43 теста прошли;
- `flutter build apk --release --dart-define=MOCK_API=false --dart-define=TEST_AUTH=false --dart-define=API_BASE_URL=https://avatracker.online/api/v1` — успешно;
- APK: `versionName=1.0.0`, `versionCode=9`;
- SHA-256 `AvaTracker-v1.0.0.apk`:
  `450a283884ed854efd946c1ac1c7d6c63b827f405a562bbe139a2068065e6abc`;
- текущая release-сборка всё ещё подписана `CN=Android Debug`, пока не
  подключён production keystore.

## [1.0.0] - 2026-07-02

### Added

- регистрация, SMS-подтверждение, вход и восстановление пароля;
- QR-отметка с геолокацией и фото-верификацией;
- профиль сотрудника, табель и аналитика опозданий;
- казахская, русская и узбекская локализации;
- поддержка телефонных кодов Казахстана и Узбекистана;
- mock-режим и локальный real-person test script;
- Android и iOS permissions, launcher icons и юридические экраны;
- unit-, localization- и widget-тесты.

### Changed

- Face verification выполняется только после сканирования QR;
- экран «Статистика» переименован в «Аналитика»;
- аналитика и текущий табель используют данные API опозданий;
- `device_id` исключён из flow отметки.

### Security

- токены и данные сессии хранятся через secure storage;
- пароли, SMS-коды и снимки лица локально не сохраняются;
- env-файлы, ключи подписи и бинарные сборки исключены из Git.
